# YumQuick — Database & Auth Setup

## Stack

| Layer          | Technology                                      |
| -------------- | ----------------------------------------------- |
| Database       | [Neon](https://neon.tech) — serverless Postgres |
| ORM / Schema   | [Drizzle ORM](https://orm.drizzle.team)         |
| Authentication | [Better Auth](https://www.better-auth.com)      |

## 1. Neon PostgreSQL

### 1.1 Create a project

1. Go to [console.neon.tech](https://console.neon.tech) and sign up / log in.
2. Click **New Project** → give it a name (e.g. `yumquick`) → **Create Project**.
3. Copy your **Connection String**:
   ```
   postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
   ```

### 1.2 Environment variable

Create a `.env` file at the project root (never commit it):

```env
DATABASE_URL=postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
BETTER_AUTH_SECRET=a-long-random-secret-string
BETTER_AUTH_URL=http://localhost:3000   # your API base URL
```

### 1.3 Install dependencies

```bash
pnpm add drizzle-orm @neondatabase/serverless
pnpm add -D drizzle-kit
```

### 1.4 Drizzle config

Create `drizzle.config.ts` at the project root:

```ts
import { defineConfig } from 'drizzle-kit';

export default defineConfig({
  schema: './db/schema.ts',
  out: './db/migrations',
  dialect: 'postgresql',
  dbCredentials: {
    url: process.env.DATABASE_URL!,
  },
});
```

### 1.5 Push schema to Neon

```bash
pnpm drizzle-kit push
```

Or generate migration files first and then apply:

```bash
pnpm drizzle-kit generate
pnpm drizzle-kit migrate
```

### 1.6 Seed the data

```bash
psql "$DATABASE_URL" -f db/seed.sql
```

Or paste `db/seed.sql` into the **SQL Editor** in the Neon console.

## 2. Better Auth

### 2.1 Install

```bash
pnpm add better-auth
```

### 2.2 Create the auth instance

Create `lib/auth.ts` (on your backend / API routes):

```ts
import { betterAuth } from 'better-auth';
import { drizzleAdapter } from 'better-auth/adapters/drizzle';
import { db } from './db'; // your Drizzle db instance
import * as schema from '../db/schema';

export const auth = betterAuth({
  database: drizzleAdapter(db, {
    provider: 'pg',
    schema: {
      user: schema.users,
      session: schema.sessions,
      account: schema.accounts,
      verification: schema.verifications,
    },
  }),
  emailAndPassword: { enabled: true },
  socialProviders: {
    google: {
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    },
  },
  secret: process.env.BETTER_AUTH_SECRET!,
  baseURL: process.env.BETTER_AUTH_URL!,
});
```

### 2.3 Expose the auth handler

Mount the handler at `/api/auth/[...all]` in your backend framework (Hono, Next.js API routes, Express, etc.):

```ts
// Hono example
import { Hono } from 'hono';
import { auth } from '../lib/auth';

const app = new Hono();
app.on(['GET', 'POST'], '/api/auth/**', c => auth.handler(c.req.raw));
```

### 2.4 React Native client

```bash
pnpm add better-auth/react-native
```

```ts
import { createAuthClient } from 'better-auth/react-native';

export const authClient = createAuthClient({
  baseURL: process.env.EXPO_PUBLIC_API_URL, // e.g. http://10.0.2.2:3000
});

// Sign up
await authClient.signUp.email({ email, password, name });

// Sign in
await authClient.signIn.email({ email, password });

// Sign out
await authClient.signOut();

// Get current session
const { data: session } = await authClient.getSession();
```

The session `user.id` is the primary key in the `users` table and FK across `orders`, `cart_items`, and `favorites`.

## 3. Database schema overview

```
── Better Auth ──────────────────────────────────────────────
users           PK: id (text, generated by Better Auth)
sessions        FK → users
accounts        FK → users (OAuth providers)
verifications   email / OTP verification tokens

── Menu ─────────────────────────────────────────────────────
categories      snacks | meal | vegan | dessert | drinks
food_items      all items; is_best_seller / is_recommended flags
promo_banners   home screen banners, linked to a food_item
topping_options Guacamole, Jalapeños, Ground Beef, Pico de Gallo
portion_options Personal, Medium, Familiar, Jumbo

── Orders ───────────────────────────────────────────────────
orders          header: status, totals, shipping address
order_items     price/name snapshot per line (topping included)
cart_items      persisted cart linked to user + food_item

── App ──────────────────────────────────────────────────────
favorites       user ↔ food_item (composite PK)
```

### Key design decisions

- Better Auth manages `users`, `sessions`, `accounts`, and `verifications` automatically — you never insert into those tables manually.
- `order_items` stores a **snapshot** of `name`, `unit_price`, `topping`, and `image_url` so historical orders are unaffected by future price changes.
- `cart_items` persists the cart server-side, enabling cross-device sync when the user is signed in.

## 4. Useful queries

```sql
-- All menu items by category
SELECT * FROM food_items
WHERE category_id = 'snacks' AND is_available = TRUE
ORDER BY name;

-- Best sellers
SELECT * FROM food_items WHERE is_best_seller = TRUE;

-- Active promo banners with food item details
SELECT pb.*, fi.name, fi.price, fi.original_price, fi.image_url, fi.rating, fi.bg_color
FROM promo_banners pb
LEFT JOIN food_items fi ON fi.id = pb.food_item_id
WHERE pb.is_active = TRUE
ORDER BY pb.sort_order;

-- Cart for a user
SELECT ci.*, fi.name, fi.price, fi.image_url, fi.bg_color
FROM cart_items ci
JOIN food_items fi ON fi.id = ci.food_item_id
WHERE ci.user_id = $1;

-- Order history for a user
SELECT o.*, json_agg(oi) AS items
FROM orders o
JOIN order_items oi ON oi.order_id = o.id
WHERE o.user_id = $1
GROUP BY o.id
ORDER BY o.placed_at DESC;

-- User favorites
SELECT fi.*
FROM favorites f
JOIN food_items fi ON fi.id = f.food_item_id
WHERE f.user_id = $1;
```
